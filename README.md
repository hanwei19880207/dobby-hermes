# Dobby-Hermes

> 京东零售质量域 AI Native 工程实践项目

## 项目定位

Dobby-Hermes 是京东零售质量域推进 **Harness 工程实践** 的核心工程仓库。项目以 AI 驱动质量工程化为目标，围绕 AI Agent / Skill / Knowledge Engineering 三大支柱，在软件工程、互联网等大型项目的质量全生命周期中落地 AI Native 方案。

命名寓意：
- **Dobby**（家养小精灵）—— AI Agent 体系，自动化执行重复性质量协调工作
- **Hermes**（信使之神 + Agent 框架）—— 知识流转与信息传递，连接质量域各角色与系统

## 核心目标

```text
传统质量工程                        →   AI Native 质量工程
──────────────────────────────────────────────────────────
人肉协调（找 5-6 人定位问题）        →   AI 智能路由 + 一键 @责任人
手工 Excel 排期追踪                  →   全景链路图 + 自动化进展预警
项目经验靠人脑记忆                   →   结构化知识工程（三层架构）
物料构造耗时 3 天                    →   SOP 沉淀 + 自动生成物料清单
问题 4 次延期才暴露                  →   实时风险感知 + 自动升级机制
新人从 0 开始                        →   知识库赋能，开箱即用
```

### 质量环节覆盖

| 环节 | AI Native 实践 | 状态 |
|------|---------------|------|
| 需求评审 | AI 参与 BRD/PRD 分析，自动识别改造范围 | 📋 规划中 |
| 测试设计 | 基于链路知识库自动生成联调用例 | 📋 规划中 |

| 物料构造 | SOP 沉淀 + 自动生成物料清单 | 📋 规划中 |
| 缺陷定位 | AI 辅助日志分析 + 智能路由找责任人 | 📋 规划中 |
| 回归验证 | 基于变更影响面自动推荐回归范围 | 📋 规划中 |
| 发布管控 | AI 辅助发布风险评估 | 📋 规划中 |
| 知识沉淀 | 项目结束自动提炼链路文档、问题案例 | 📋 规划中 |

## 分阶段路线图

### 阶段一（0-2 月）：基建 + 主测试 Agent MVP
- [ ] 工程骨架初始化
- [ ] AI 主测试 Agent 上线（找人/链路全景/日报/预警）
- [ ] 首批 10 个核心系统链路图谱构建
- [ ] 飞书 Bot 双向通信打通

### 阶段二（2-4 月）：技能扩展 + 知识沉淀
- [ ] 物料 SOP 标准化
- [ ] 联调用例自动生成
- [ ] 风险自动升级机制
- [ ] 知识库 L1 层（链路层）构建

### 阶段三（4-6 月）：全链路覆盖 + 自学习
- [ ] 全链路质量环节覆盖
- [ ] Agent 自学习迭代
- [ ] 知识库 L2/L3 层构建
- [ ] 协调工作量减少 60% 目标验证

### 中期（6-12 月）：可视化 + 标准化
- [ ] 跨域协作可视化（依赖图谱/数据标识追踪）
- [ ] 主测职责标准化
- [ ] 质量域 AI Native 最佳实践输出

### 长期（12 月+）：知识工程规模化
- [ ] 知识闭环机制（采集→加工→审核→发布→消费→校准）
- [ ] 新项目自动推荐标准链路
- [ ] 变更影响面自动分析
- [ ] 主测从「协调者」转型「质量攻坚者」

## 技术栈

| 层级 | 技术选型 |
|------|---------|
| Agent 框架 | Hermes Agent（多模态 AI Agent 运行时） |
| 大模型 | DeepSeek / Claude / GPT 多模型接入 |
| 即时通讯 | 飞书 Bot（消息/文档/审批/日程） |
| 知识存储 | Markdown + Git 版本化知识库 |
| 链路图谱 | Mermaid / Graphviz 可视化 |
| CI/CD | GitHub Actions |
