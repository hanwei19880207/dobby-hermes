# Oxygen SE 体系

> 来源：D端AI例会 WK22 + WK23 | 最后更新：2026-06-07

## 概述

Oxygen SE 是零售产研中心 D 端 AI Native 的核心工程体系，分为 Zero（设计侧）和 One（产品/架构推演侧）。

## Zero（设计侧）

定位：AI 时代的设计系统——不再是给人看的规范库，而是让 AI 能够理解、调用、推理和执行的体验基础设施。

关键能力：
- 产品试点（我京）：PRD 上下游依赖分析、专属知识库、PRD 预审 → 提效 ~30%
- 设计试点：Design-to-MD Skill、设计审查 Skill
- 平台能力：技能市场、知识提炼、对话记忆、Token 消耗提醒、Zero CLI

## One（产品/架构推演侧）

### 智能体技术架构

```
交互层 → 智能体核心（Planning & Reasoning）→ 智能体基建
                                                    ├── 向量 RAG 召回
                                                    ├── 图 RAG 召回
                                                    ├── SFT 后训练（远期）
                                                    └── 模型网关 → LLM 基础模型
```

核心 Agent：Claude Code / Codex（可插拔可替换）

### 上工程 — Hermes 自学习机制

```
操作轨迹记录 → 信号整合（显式+隐式）→ 奖惩分析 → 知识蒸馏 → 定期学习
                                                                        ↓
                                                             生成/更新 SKILL.md 和元数据
                                                                        ↓
                                                             知识自更新闭环
```

### 下工程 — Harness 架构

```
全局约束（中间件/高可用/配置规范）
        +
开发验证（Agent开发 → 自动测试 → 智能Debug → 代码修复 → 再测试）
        +
基础环境（本地/特性/基线测试环境）
```

模型：可插拔可替换（Claude Code ⇄ Codex ⇄ 其他 LLM）

### 知识蒸馏策略

1. 产品/架构领域划分（人工梳理一次性投入）
2. 导入原始知识素材（PRD/TRD/楼层组件说明 + 代码/Wiki）
3. LLM 持续蒸馏知识 → 各领域规则 Skill
4. 消费知识推理架构（ONE Agent）
5. 工程实现反馈回流 → 自学习机制

蒸馏模型：DeepSeekV4-pro / GLM-5.1

## 知识体系设计

- **端**：APP/小程序/PC，按页面、楼层组织
- **服务端**：按业务域/子域组织（交易、财经、流量场域、订单中心...）
- **每层**：产品（职责边界+功能描述+PRD）+ 技术（系统架构+领域知识+TRD）+ 设计（设计稿+设计原则）
- **全局**：设计规范、资损规范、高可用规范、跨域边界冲突处理
