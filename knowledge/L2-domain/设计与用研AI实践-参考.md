# 设计与用研团队 AI 实践（参考案例）

> 来源：D端AI例会 WK23 分享 | 最后更新：2026-06-07

## 为什么值得学习（老板好评）

1. 从设计哲学到落地的完整链路清晰
2. .md → Skill 的标准化路径可复制、可规模化
3. 有明确的组织机制（共建→评审→发布→复用→反馈→迭代）
4. 核心金句：**"你今天用AI节省的每一分钟，都不如你今天沉淀的一个Skill来得有价值"**

## 核心理念

从"传统设计系统（面向人）"升级为"AI 时代设计系统（面向 AI + 人）"——

| 维度 | 传统 | AI时代 |
|------|------|--------|
| 编写方式 | 偏视觉展示 | 结构化、语义化、可解析 |
| 内容重点 | "长什么样" | "为什么这样设计、何时使用、如何推导" |
| 经验沉淀 | 分散在人和项目中 | 结构化沉淀为组织知识资产 |
| 生命周期 | 相对静态 | 持续自更新/自查/自演化 |
| 最终定位 | 组件与规范库 | AI Native Experience Infrastructure |

## 落地路径

```
设计哲学 → 设计系统(16.0) → 产品场域组件化 → Design Wiki → Skill 沉淀 → AI 应用 → 组织机制
```

## Design Wiki 架构

6 大 Zone 架构，已沉淀 180+ .md 文档

## Skill 体系

| 类别 | 数量 | 说明 |
|------|------|------|
| 基底类 | 3 | Zero→.MD、.MD→HTML门户 |
| 通用类 | 1 | 设计稿规范审核 |
| 场景类 | 31 | 审核提效(7) + 交付提效(6) + 生成提效(13) + 分析提效(4) |
| 机制类 | 规划中 | 共建机制 |

## 关键成果

- 逛底导审核机器人：615 次审核，效率提升 90%
- 大促分会场 AI 工作流：覆盖全年 400+ 个 S 级大促分会场
- 营销会场 AI 工作流：覆盖全年 15 场 A/B 级 + 239 场业务会场
- 竞品截图：手机截图自动同步 Mac + 自动归类分析

## 组织机制

- 第一阶段：各团队按需充分尝试
- 第二阶段：建立能力社区，让优质能力成为团队公共资产
- 核心机制：**Skill 共建 → 评审 → 发布 → 复用 → 反馈 → 迭代**

## 对质量域的启发

1. **知识工程路径**：设计能做到 180+ .md 规范文档，质量域同样需要系统化的测试规范/用例/案例知识库
2. **Skill 标准化**：基底类→通用类→场景类的分层设计值得借鉴
3. **组织机制**：从"个人实践"到"团队资产"的转化机制
4. **度量思维**：每个 Skill 都有明确的效果数据（审核次数、效率提升%）
