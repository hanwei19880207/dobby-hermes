    ├── 下工程——Harness架构

### Hermes + Harness 架构（关键）

**上工程 Hermes**：持续记录操作轨迹 → 显式信号(采纳/拒绝)+隐式信号(编辑/增删) → 奖惩模型 → 自动生成/更新 SKILL.md 和元数据 → 知识自进化

**下工程 Harness**：
- 全局约束：中间件规范、高可用规范、配置规范
- 开发验证：Agent开发 → 自动测试 → 智能Debug → 代码修复 → 再测试闭环
- 基础环境：本地测试环境 / 特性测试环境 / 基线测试环境

## 三、两次分享剖析

### WK22 — 营销与用户技术部

**核心逻辑**：中后台标准页面 → Page Spec 协议 → 受控 AI Coding

```
避免AI裸写代码 → 让AI读取需求 → Page Spec标准化
→ 组件库MCP约束 → 代码上下文理解 → 质量门禁
```

**关键数字**：
- AI Coding 代码覆盖率 63.42%（B端76.55%，C端53.97%）
- 沉淀 77 个 Skills（8基础 + 31通用 + 27营销 + 8会员 + 3用户）
- 近30天 Token 消耗 353 亿（环比↑80%）

**启发**：他们把 Skills 作为可沉淀、可度量、可复制的组织资产来管理

### WK23 — 设计与用研部（老板好评）

**核心逻辑**：设计系统从"给人看"升级为"让AI能理解、推理、调用、执行"

```
设计哲学 → 设计系统(16.0) → 产品场域组件化 
→ Design Wiki → Skill 沉淀 → AI 应用 → 组织机制
```

**Design Wiki 6大Zone架构**：从规范到组件到场景到案例的结构化知识体系

**四大类Skill**：基底类(3) + 通用类(1) + 场景类(31) + 机制类(规划中)

**关键数字**：
- 180+ .md 规范文档
- 100+ 模块资产（40+通用楼层 + 20+垂类 + 20+弹层）
- 逛底导审核机器人：615次审核，效率提升90%
- 营销会场AI工作流：覆盖全年400+S级大促分会场

**老板好评的原因推测**：
1. 从设计哲学到落地的完整链条清晰
2. .md → Skill 的标准化路径可复制
3. 有明确的组织机制（共建→评审→发布→复用→反馈→迭代）
4. 一句话金句："你今天用AI节省的每一分钟，都不如你今天沉淀的一个Skill来得有价值"
